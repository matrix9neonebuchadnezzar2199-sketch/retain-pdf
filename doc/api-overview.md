# サービス概要

本ドキュメントは、本プロジェクト API の全体構造を説明します。

## 1. サービスポート

- フロントエンド静的ページのデフォルト: `http://127.0.0.1:8080`
- Rust API のデフォルト: `http://127.0.0.1:41000`
- 簡易同期 API のデフォルト: `http://127.0.0.1:42000`
- ヘルスチェック: `GET /health`
- 業務プレフィックス: `/api/v1`

## 2. メインチェーン

現在のメインチェーン:

1. PDF をアップロード
2. メインタスク `POST /api/v1/jobs` を作成
3. メインタスク内部で OCR サブタスク `{job_id}-ocr` を派生
4. OCR 完了後に標準化 `document.v1` を生成
5. 翻訳とレンダリングへ進む
6. PDF / Markdown / ZIP をダウンロード

## 3. 統一レスポンス形式

成功:

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

失敗:

```json
{
  "code": 400,
  "message": "具体的なエラーメッセージ"
}
```

## 4. 現在のメイン provider

- 本番メインラインは `mineru` が中心
- `paddle` は接続済みだが、開発・デバッグ用途が主
